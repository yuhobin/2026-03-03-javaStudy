# 📅 2026-03-20 Java_OOP_Access Modifier

### 🏷️ Topic
객체지향 프로그램의 접근지정어와 제어어 및 크롤링 실습

### 📝 What I Learned
객체지향의 4대 특성 중 하나인 캡슐화를 구현하기 위해 접근지정자의 종류와 범위를 공부했다. 
모든 곳에서 접근 가능한 public, 동일 패키지와 상속 관계까지 허용되는 protected, 동일 패키지 내에서만 가능한 default, 그리고 클래스내부에서만 접근할 수 있는 private의 차이를 명확히 확인했다. 
데이터 보호와 무결성을 위해 변수는 private으로 설정하고, 외부에서 안전하게 접근할 수 있도록 Getter와 Setter 메소드를 활용하는 방식을 익혔다.
이를 응용해 음원사이트 크롤링 실습을 진행하며 가져온 데이터를 Music 객체의 각 변수에 할당하고 제어하는 실무적인 과정도 함께 체득했다.

### 💡 Note
클래스와 메소드는 외부 노출을 위해 주로 public을 쓰지만 변수는 보안을 위해 반드시 private을 사용하고 메소드를 거쳐 접근해야 한다. 
특히 크롤링 데이터를 다룰 때 객체가 생성되지 않은 null 상태에서 접근하면 NullPointerException이 발생할 수 있으므로 항상 유효성 체크를 병행하는 습관을 가져야 한다.
